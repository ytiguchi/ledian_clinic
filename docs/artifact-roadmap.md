# アーティファクトロードマップ

共通データソースを中心に、各チャネル向けのアウトプットを段階的に整備します。

## フェーズ1: 基盤整備（進行中）
- [x] ディレクトリ構造の確立 (`data/`, `docs/`, `apps/`、チャネル別 content 階層)。
- [x] D1 環境整備（internal prod のみ、public prod & stg、マイグレーション適用）。
- [ ] 正規化データのフィールド定義を YAML で確定（`catalog/`, `pricing/` の必須/任意項目）。
- [ ] サンプルデータ投入（1カテゴリ/1施術/複数プランで最低限のパスを通す）。

## フェーズ2: データ投入とバリデーション
- [ ] 全カテゴリ・施術・プランのマスタを `catalog/`, `pricing/` に入力。
- [ ] バリデーションスクリプト作成（必須フィールド検知、価格の整合性、slug 重複チェック）。
- [ ] DB 反映用の変換/ロードスクリプト雛形（CSV/SQL 生成）。
- [ ] internal→public のマスク/抽出スクリプト（`docs/seed-and-sync.md` に沿って実装）。

## フェーズ3: チャネル別アウトプット
- [ ] Web: LP/料金表用のコピー、FAQ、導線案を `data/content/web/` に記述。静的ページ or CMS 取り込み用フォーマット決定。
- [ ] 社内Bot: 料金・施術説明・注意事項を `data/content/bot/` で Q&A 形式に整理。シナリオ/インテント表現を揃える。
- [ ] カウンセリング資料: 説明文/禁忌/術後ケアを `data/content/counseling/` に整理し、資料テンプレート（スライド or PDF生成）への流し込みを定義。

## フェーズ4: 自動出力と配布
- [ ] `scripts/` 配下でチャネル別ビルド（例: Web 向け静的 JSON、Bot 向けナレッジ CSV、Counseling 資料 PDF）。
- [ ] 生成物を `outputs/`（gitignore 想定）に集約し、配布/配信フローを決定。
- [ ] 定期更新フロー（レビュー手順、リリースノート、周知チャネル）を運用化。
